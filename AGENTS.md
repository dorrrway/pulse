# AGENTS.md

## 工程原则

1. 代码语言优雅简洁，现代健壮。
2. 避免 workaround。
3. 符合 macOS 最新规范和官方推荐语义。
4. 避免临时、敷衍的解决方案，避免埋下隐患。

## 发布规则

所有提交、推送、发布前必须遵守 [docs/RELEASE_RULES.md](docs/RELEASE_RULES.md)。

尤其注意：

- 任何用户可见能力、文案、隐私边界、安装方式、运行方式或发布产物变化，都要同步检查 README、英文 README、隐私协议和更新日志。
- 涉及读取、存储、传输、权限、沙盒、登录项、进程信息、网络或系统数据的变化，必须先更新隐私说明，再推送。
- 发布前必须检查工作区是否混有无关改动，只能显式暂存本次范围内的文件。
- 不要把本地机器状态、Xcode 用户数据、DerivedData、临时日志、截图草稿或未确认实验改动混进发布提交。
