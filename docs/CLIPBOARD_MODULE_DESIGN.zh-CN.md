# 剪贴板模块设计说明

最后更新：2026 年 5 月 26 日

## 目标

在 Pulse Island 现有“资源监控”和“应用程序”之外，新增第三个独立模块：“剪贴板”。模块只接入 Island 的功能切换，不改变已有资源监控、应用程序、固定面板和更新器的数据流或样式。

剪贴板模块用于查看本机剪贴板历史，区分文本、链接、文件和图片内容，保留原始 pasteboard 表示用于完整复制回系统剪贴板，并记录来源 App 线索与敏感/临时 marker。所有数据处理只在本机完成，不上传。

## 第一版范围

- 新增 Pulse Island 模块：剪贴板。
- App 启动后默认监听系统剪贴板变化，不提供关闭剪贴板模块的主开关。
- 读取由 macOS 系统权限控制；读取失败或被拒时显示状态和恢复指引，不清空既有历史。
- 保存文本、链接、文件、图片和原始 pasteboard type/data 表示。
- 文件复制按 pasteboard 中的文件 URL/路径表示识别和保存，不主动读取或复制文件本体内容。
- 敏感/临时 marker 不跳过：全部记录、可复制、可搜索。
- 列表默认遮罩敏感内容；Copy 按钮直接复制完整原始 payload，不要求先 reveal。
- 从剪贴板历史中点击记录只写回系统剪贴板，不再生成新的历史记录；双击记录会在写回系统剪贴板后，通过 macOS 辅助功能权限向当前聚焦位置发送一次粘贴快捷键；外部 App 再次复制相同 payload 时，按 fingerprint 合并到同一条记录并移动到列表顶部。
- 支持搜索、复制、展开查看、删除单条和清空历史。
- 历史跨重启持久保存到本机 Application Support，默认不按条目数量清理，只按 30 天保留；设置中保留条数可选择 100、500、1000 或无限制，保留时间可选择无限制、7 天、30 天或 90 天。
- 图片显示较大的原图预览并限制最大高度。
- OCR 默认关闭；开启后使用本机 Vision 生成搜索文本并写入本地索引字段。

## 监听和授权

监听采用轻量轮询：`ClipboardHistoryController` 每 0.5 秒检查 `NSPasteboard.general.changeCount`，只有变化时读取内容。这样避免持续解析剪贴板，也不依赖后台全局事件。

`ClipboardPasteboardClient` 封装 AppKit 读写，便于单元测试使用 fake pasteboard client，避免污染真实系统剪贴板。读取时保存当前 macOS pasteboard access behavior 和最近一次失败原因；UI 根据这些状态提示用户到系统设置恢复授权。

## 数据模型

每一次剪贴板变化生成一条 `ClipboardHistoryEntry`，entry 下包含一个或多个 `ClipboardHistoryItem`。模型字段包含：

- change count 和采集时间。
- 声明来源 App 与推断来源 App，并区分二者。
- entry 级和 item 级 marker types。
- 每个 item 的内容类型：文本、链接、文件、图片或混合。
- 展示文本、搜索文本、OCR 文本。
- 原始 pasteboard representations 的 blob 引用。

原始数据不直接塞入列表模型，统一写入 blob 目录；metadata 只保存 entry、item、marker、source、搜索字段和 blob id。

## 敏感和临时 marker

剪贴板 item 可能带有 marker 类型，用于表达内容敏感、临时、自动生成或来自远程剪贴板。Pulse 的策略是记录这些 marker，但不让 marker 阻止采集。

第一版识别并展示的常见 marker 包括：

- `org.nspasteboard.ConcealedType`
- `org.nspasteboard.TransientType`
- `org.nspasteboard.AutoGeneratedType`
- `org.nspasteboard.source`
- remote clipboard/source 相关类型

处理原则：

- 带 marker 的内容仍然保存、搜索、OCR 和复制。
- `ConcealedType`、`TransientType`、`AutoGeneratedType` 和 legacy transient 类型会让记录默认遮罩。
- 遮罩只影响 UI 展示，不影响 Copy 按钮复制完整原始 payload。
- Reveal 状态只保存在当前 UI 会话，不写入磁盘。

常见场景：

- 密码管理器复制密码。
- 复制一次性验证码。
- 复制 API key、访问令牌、密钥等敏感文本。
- 某些 App 从安全输入框复制内容。
- App 内部生成的一次性粘贴内容。
- Universal Clipboard 或远程剪贴板同步内容。

## 搜索口径

第一版只做本地文本层面的搜索：

- 顶部分类筛选支持全部、文本、图片、链接和文件；混合记录只要包含对应 item，就会出现在对应分类里。
- 文本项：搜索原始文本内容。
- 链接项：搜索 URL 字符串和可展示标题字段。
- 文件项：搜索文件名或路径展示文本。
- 图片项：默认只按文件/类型等元数据搜索。
- 图片项如启用 OCR：搜索 OCR 得到的文字。
- 敏感项参与搜索，但搜索结果仍默认遮罩。

图片语义搜索，例如按“猫”“表格”“蓝色按钮”等视觉含义搜索，不进入第一版范围。

## 图片展示口径

图片项不使用无意义的小缩略图。剪贴板里的图片，尤其是截图，只有足够大的预览才有实际识别价值。

展示方式：

- 在列表中按面板宽度显示较大的图片预览。
- 保持原始比例。
- 对超高图片限制最大高度，避免单条记录占满整个面板。
- 展开后显示更完整的内容摘要。

## 来源 App 归因

来源 App 归因分为两类：

- 声明来源：优先读取 pasteboard source marker，例如 `org.nspasteboard.source`。
- 推断来源：没有声明来源时，在 `changeCount` 变化时读取当时的前台 App。

推断来源字段包括 App 名称、bundle identifier 和 bundle path。UI 图标通过 bundle path 使用 `NSWorkspace.shared.icon(forFile:)` 获取，并在视图层缓存。

推断来源不是系统保证的绝对来源。用户手动复制时通常可靠；如果后台 App 程序化写入剪贴板，可能会归因到当前前台 App。

## 隐私和存储口径

剪贴板历史属于高敏感数据。实现和发布必须同步检查 README、英文 README、隐私政策和更新日志。

第一版策略：

- 默认启用剪贴板历史。
- 所有剪贴板历史只保存在本机。
- 不上传剪贴板内容、OCR 结果、来源 App 信息、marker 或图片。
- 默认不按条目数量清理，只按 30 天保留；设置中保留条数可选择 100、500、1000 或无限制，保留时间可选择无限制、7 天、30 天或 90 天。
- 提供一键清空，清空会删除 metadata、blob、OCR 文本和内存缓存。
- OCR 默认关闭；开启后只使用本机 Vision。
- 设置页只提供保留条数、保留时间和 OCR 开关，不提供关闭剪贴板模块的主开关。

## 实现模块

- `ClipboardHistoryController`：监听 `NSPasteboard.general.changeCount`，在变化时读取剪贴板并更新授权状态。
- `ClipboardHistoryStore`：管理列表、搜索、复制、清空、保留策略和持久化。
- `ClipboardHistoryPersistence`：管理本地 JSON metadata 和 blob 文件目录。
- `ClipboardPasteboardClient`：封装 AppKit pasteboard 读写，支持 fake client 测试。
- `ClipboardFocusedPasteCommandPoster`：封装辅助功能授权检查和单次 Command-V 发送，只用于双击直接粘贴。
- `ClipboardOCRService`：可选 OCR 服务，使用本机 Vision 框架。
- `ClipboardPanelView`：剪贴板模块 UI。

## 验证范围

- marker 不跳过，敏感项遮罩但可复制。
- 原始 pasteboard representations 可恢复。
- 搜索可命中遮罩项。
- 保留条数和保留天数会清理过期 metadata 与 blob。
- 清空历史会删除 blob。
- 现有 Island 模块切换、Installed Apps、Resource Monitor、语言/设置测试继续通过。
