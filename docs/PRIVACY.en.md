Pulse is a local Dynamic Island-style Mac utility. It is designed to show system status, the Applications list, clipboard history, screenshot tools, and Bluetooth devices on the Mac where it is running.

## Data Pulse Reads

Pulse may read local data including:

- CPU usage
- Memory usage
- Disk capacity and disk I/O counters
- Network byte counters
- Battery status
- Thermal state
- The system's last boot time, used to calculate system runtime
- Names of running processes and their CPU and memory usage
- Local app bundle paths used to display app icons
- App names, bundle IDs, versions, and bundle paths from standard application locations, plus those apps' running state, used to show the Applications list
- Local Bluetooth device names, types, connection state, Bluetooth addresses, and battery or charging-state fields exposed by macOS after the user grants Bluetooth access, used to show the Bluetooth device list, connect or disconnect paired devices, display readable battery levels and charging state, and show local low-battery alerts for Bluetooth devices; AirPods-style devices show left, right, and case battery levels when macOS returns those fields
- System clipboard changes and contents, including text, links, file URL/path representations, images, original pasteboard representations, sensitive or transient markers, and source-app information declared by the clipboard or inferred by Pulse, used to show Clipboard history; when files are copied, Pulse does not proactively read or store the file contents themselves
- Full-screen, window, or custom selection pixels through the native macOS screenshot flow when the user explicitly triggers the Screenshots module; the capture is written to the system clipboard

System metrics, processes, the Applications list, and the Bluetooth device list are used only to render the app interface. They are not uploaded or used for persistent tracking. The Applications list and running state are refreshed on demand in local memory and are not written to a persistent database. After Bluetooth access is granted, the Bluetooth device list is refreshed at a low frequency in local memory while Pulse is running so the Bluetooth panel and low-battery alerts remain available; the refresh frequency increases while the Bluetooth panel is open. Pulse does not save Bluetooth addresses, device serial numbers, battery history, or charging history. App bundle paths added by the user to Favorite Apps are saved in local preferences so Pulse can restore the Favorite Apps panel next time.

Clipboard history is enabled by default and is stored in the local Application Support directory after macOS allows clipboard reads. Pulse does not automatically prune by entry count by default and keeps history for 30 days; users can choose a 100, 500, 1000, or unlimited entry limit in Settings, choose an unlimited, 7-day, 30-day, or 90-day retention time, and clear history at any time. Items marked with `org.nspasteboard.ConcealedType`, `org.nspasteboard.TransientType`, `org.nspasteboard.AutoGeneratedType`, remote clipboard/source markers, and similar markers are still recorded. These markers do not prevent storage, search, or copy; they only make the interface mask more sensitive entries by default. Direct paste from a double-clicked clipboard entry first writes that entry back to the system clipboard, then uses macOS Accessibility permission to send one paste shortcut to the currently focused target; Pulse does not read the target app's input contents, window contents, or accessibility tree. OCR is off by default; when enabled, image text recognition uses the local Vision framework, and OCR text is stored only in the local search index.

The screenshot completion preview exposes user-triggered Save, Share, Edit, Pin, and Recognize Text actions. Save writes the current screenshot as a PNG only after the user chooses a location. Share hands the screenshot to the macOS share sheet, where the user chooses AirDrop, Mail, Messages, or another system share destination. Edit processes the current screenshot only in a local foreground floating editor, can add brush-based mosaic redaction, rectangles, circles, arrows, pen strokes, and text; the editor's Save, Share, and Pin actions also operate only on the current edited image, and completing the edit writes the edited image back to the system clipboard. It does not upload the image or automatically save another file. Pin keeps the current screenshot in a local foreground floating window until the user closes that window; it does not upload the image or write another file. Recognize Text uses only the local Vision framework; recognized text is not uploaded and does not replace the clipboard automatically. It is written to the system clipboard only if the user chooses Copy Text in the result window. When the user drags the screenshot preview to Finder, chat windows, or documents, Pulse provides PNG data for that drag and prepares a PNG file representation in the system temporary directory. Old temporary drag files are cleaned up by later drags.

The Screenshots module runs only when the user clicks a screenshot action or triggers its shortcut, and first use requires macOS Screen Recording permission. Pulse hides its island panel by default before calling the native macOS `screencapture` tool and writing the result to the system clipboard; users can turn off Hide Pulse while capturing in the Screenshots panel when they want Pulse itself to appear in the image. If Clipboard history is enabled, that screenshot is stored as a local image entry under the existing Clipboard history rules. Pulse does not upload screenshots to the network. If permission has not been granted or was denied, Pulse requests access or opens the system privacy settings.

## Data Pulse Does Not Collect

Pulse does not collect, store, or transmit:

- Personal files
- Contacts, photos, calendars, or location data
- Keychain contents; if the user copies passwords, one-time codes, API keys, or other credentials to the system clipboard, the Clipboard module stores them locally as history and masks them by default in the list
- In-app behavior analytics or feature-usage telemetry
- Persistent records or uploads of Bluetooth addresses, device serial numbers, or persistent tracking identifiers

## Network Usage

Pulse requests the TimeLikeSilver-hosted appcast over HTTPS to check whether a new version is available. When the user clicks Update, Pulse downloads the matching release archive and Sparkle verifies its signature before installation.

As with ordinary HTTPS requests, the appcast request may be processed with standard request metadata such as IP address, user agent, request time, and requested path. TimeLikeSilver may use these requests for security, operations, and aggregate update-check counts to understand Pulse runtime trends. Update checks do not attach CPU, memory, device model, system boot time, process lists, application lists, Bluetooth device lists, Bluetooth battery levels, Bluetooth charging state, clipboard contents, app bundle paths, file contents, or Sparkle system profiling data.

## Local Preferences

Pulse may store local preferences, such as language, appearance, launch-at-login, Applications display mode, Favorite Apps bundle paths, the Clipboard OCR setting, the Clipboard entry limit, the Clipboard retention-day limit, whether Pulse hides itself during screenshots, and the shortcuts for waking Clipboard, waking Applications, and the three screenshot modes, using macOS local storage. The Bluetooth device list, connection state, battery levels, and charging state are not written to local preferences.
