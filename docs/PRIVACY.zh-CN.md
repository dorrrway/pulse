Pulse 是一款本地运行的 macOS 工具，会像灵动岛一样停留在屏幕顶部，用于展示当前 Mac 的系统状态、应用程序列表和剪贴板历史。

## Pulse 会读取的数据

Pulse 可能会读取以下本机数据：

- CPU 使用情况
- 内存使用情况
- 磁盘容量与磁盘 I/O 计数
- 网络收发字节计数
- 电池状态
- 热状态
- 系统上次启动时间，用于计算开机时长
- 正在运行进程的名称及其 CPU、内存占用
- 用于显示 App 图标的本机 App bundle 路径
- 标准应用位置中的 App 名称、bundle id、版本和 bundle 路径，以及这些 App 的运行状态，用于显示“应用程序”列表
- 系统剪贴板变化及其内容，包括文本、链接、文件 URL/路径表示、图片、原始 pasteboard 表示、敏感或临时 marker，以及剪贴板声明或 Pulse 推断的来源 App 信息，用于显示“剪贴板”历史；复制文件时 Pulse 不主动读取或保存文件本体内容

系统指标、进程和应用程序列表只用于显示应用界面，不会上传或用于持久追踪。“应用程序”列表和运行状态只在本机内存中按需刷新，不会写入持久数据库。用户加入“常用应用”的 App bundle 路径会保存在本机偏好设置中，用于下次打开时恢复常用应用面板。

剪贴板历史默认启用，并在 macOS 允许读取后保存在本机 Application Support 目录中。Pulse 默认不按条目数量自动清理，只按 30 天保留历史；用户可在设置中选择 100、500、1000 或无限制的保留条数，也可选择无限制、7 天、30 天或 90 天的保留时间，并可一键清空。带有 `org.nspasteboard.ConcealedType`、`org.nspasteboard.TransientType`、`org.nspasteboard.AutoGeneratedType`、remote clipboard/source 等 marker 的内容也会记录；这些 marker 不会阻止保存、搜索或复制，只会让界面默认遮罩更敏感的记录。双击剪贴板记录的“直接粘贴”会先把该记录写回系统剪贴板，再通过 macOS 辅助功能权限向当前聚焦位置发送一次粘贴快捷键；Pulse 不读取目标 App 的输入内容、窗口内容或辅助功能树。OCR 默认关闭；开启后，图片文字识别只使用本机 Vision 框架，OCR 文本也只保存在本机历史索引中。

## Pulse 不会收集的数据

Pulse 不会收集、存储或传输：

- 个人文件
- 通讯录、照片、日历或定位数据
- 钥匙串内容；如果用户把密码、验证码、API key 或其他凭据复制到系统剪贴板，剪贴板模块会按历史记录本机保存，并在列表中默认遮罩
- 应用内行为分析数据或功能使用遥测
- 设备序列号或持久追踪标识

## 网络使用

Pulse 会通过 HTTPS 请求 TimeLikeSilver 官网的 appcast 文件，用于检查是否存在新版本；当用户点击更新时，Pulse 会下载对应发布包并由 Sparkle 验证签名后安装。

和普通 HTTPS 请求一样，appcast 请求可能会由服务器处理标准请求元数据，例如 IP 地址、User-Agent、请求时间和请求路径。TimeLikeSilver 可能会将这些请求用于安全、运维和汇总更新检查次数，以了解 Pulse 的运行趋势。更新检查不会附加 CPU、内存、设备型号、系统启动时间、进程列表、应用程序列表、剪贴板内容、App bundle 路径、文件内容或 Sparkle 系统画像。

## 本地偏好设置

Pulse 可能会使用 macOS 本地存储保存语言、外观、开机启动、应用程序列表显示方式、常用应用 App bundle 路径、剪贴板 OCR 开关、剪贴板保留数量和剪贴板保留天数等偏好设置。
