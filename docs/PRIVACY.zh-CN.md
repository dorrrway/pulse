Pulse 是一款本地运行的 macOS 工具，会像灵动岛一样停留在屏幕顶部，用于展示当前 Mac 的系统状态、应用程序列表、剪贴板历史、备忘和待办、截屏录屏工具和蓝牙设备列表。

## Pulse 会读取的数据

Pulse 可能会读取以下本机数据：

- CPU 使用情况
- 内存使用情况
- 磁盘容量与磁盘 I/O 计数
- 网络收发字节计数
- 电池状态
- 热状态
- 系统上次启动时间，用于计算开机时长
- 正在运行进程的名称及其 CPU、内存占用
- 用于显示 App 图标的本机 App bundle 路径
- 标准应用位置中的 App 名称、bundle id、版本和 bundle 路径，以及这些 App 的运行状态，用于显示“应用程序”列表
- 用户授予蓝牙权限后，本机蓝牙设备的名称、类型、连接状态、蓝牙地址，以及 macOS 暴露的电量和充电状态字段，用于显示蓝牙设备列表、连接/断开已配对设备、展示可读取的电量和充电状态，并在设备电量过低时显示本机提醒；AirPods 类设备在 macOS 返回对应字段时会分别显示左耳、右耳和盒子电量
- 系统剪贴板变化及其内容，包括文本、链接、文件 URL/路径表示、图片、原始 pasteboard 表示、敏感或临时 marker，以及剪贴板声明或 Pulse 推断的来源 App 信息，用于显示“剪贴板”历史；复制文件时 Pulse 不主动读取或保存文件本体内容
- 用户主动输入到“备忘”模块的备忘文本、待办完成状态、置顶状态、创建时间和更新时间
- 用户主动触发“截屏录屏”模块时，由 macOS 原生截图能力读取全屏、窗口或自定义区域画面，并把截图写入系统剪贴板
- 用户主动从“截屏录屏”模块开始录屏时，由 macOS 原生 ScreenCaptureKit 读取全屏、窗口或自定义区域的视频帧；录屏不采集系统音频或麦克风音频

系统指标、进程、应用程序列表和蓝牙设备列表只用于显示应用界面，不会上传或用于持久追踪。“应用程序”列表和运行状态只在本机内存中按需刷新，不会写入持久数据库。授予蓝牙权限后，蓝牙设备列表会在 Pulse 运行期间于本机内存中低频刷新，用于保持蓝牙面板和低电量提醒可用；蓝牙面板打开时会提高刷新频率。Pulse 不会保存蓝牙地址、设备序列号、电量历史或充电历史。用户加入“常用应用”的 App bundle 路径会保存在本机偏好设置中，用于下次打开时恢复常用应用面板。

剪贴板历史默认启用，并在 macOS 允许读取后保存在本机 Application Support 目录中。Pulse 默认不按条目数量自动清理，只按 30 天保留历史；用户可在设置中选择 100、500、1000 或无限制的保留条数，也可选择无限制、7 天、30 天或 90 天的保留时间，并可一键清空。带有 `org.nspasteboard.ConcealedType`、`org.nspasteboard.TransientType`、`org.nspasteboard.AutoGeneratedType`、remote clipboard/source 等 marker 的内容也会记录；这些 marker 不会阻止保存、搜索或复制，只会让界面默认遮罩更敏感的记录。双击剪贴板记录的“直接粘贴”会先把该记录写回系统剪贴板，再通过 macOS 辅助功能权限向当前聚焦位置发送一次粘贴快捷键；Pulse 不读取目标 App 的输入内容、窗口内容或辅助功能树。OCR 默认关闭；开启后，图片文字识别只使用本机 Vision 框架，OCR 文本也只保存在本机历史索引中。

备忘模块只保存用户主动输入的内容。普通备忘和待办会保存在本机 Application Support 目录中的 JSON 文件里，用于下次打开 Pulse 时恢复列表、筛选、搜索、完成状态和置顶状态。复制备忘会把该条文本写入系统剪贴板；如果剪贴板历史开启，这次复制会按现有剪贴板规则进入本机历史。Pulse 不会上传备忘内容，不会同步到 iCloud、日历或提醒事项，也不会为备忘创建通知。

截图完成后的预览提醒会提供用户主动触发的保存、分享、编辑、图钉和识别文字动作。保存动作只会在用户选择位置后把当前截图写为 PNG 文件；分享动作会把截图交给 macOS 系统分享面板，由用户选择 AirDrop、邮件、信息或其他系统分享目标；编辑动作只会在本机前台悬浮编辑窗口中处理当前截图，可用笔触涂抹马赛克，并叠加矩形、圆形、箭头、画笔和文字；编辑器内的保存、分享和图钉也只作用于当前编辑后的图片，完成后会把编辑后的图片重新写入系统剪贴板，不会上传或自动另存文件；图钉动作只会把当前截图保留在本机前台悬浮窗口中，直到用户关闭该窗口，不会额外上传或写入文件；识别文字动作只使用本机 Vision 框架，识别结果不会上传，也不会自动替换剪贴板内容，只有用户在结果窗口中选择复制文字时才会写入系统剪贴板。用户把截图预览拖到 Finder、聊天窗口或文档时，Pulse 会为这次拖拽提供 PNG 数据，并在系统临时目录准备一个 PNG 文件表示；旧的临时拖拽文件会在后续拖拽时清理。

截屏录屏模块只在用户点击截图或录屏动作、或触发截图或录屏快捷键时运行，首次使用需要授予 macOS“屏幕录制”权限。截图时，Pulse 默认会在调用 macOS 原生 `screencapture` 前临时隐藏自己的灵动岛面板；用户也可在面板底部关闭“隐藏 pulse 界面”，让截图包含 Pulse 本身。截图会写入系统剪贴板；若剪贴板历史处于启用状态，这张截图会作为图片记录进入本机剪贴板历史。录屏开始前，全屏和窗口录屏会先复用原生截图选择器生成一张临时选择截图，只读取系统写入的目标范围元数据后立即删除；自定义区域录屏会显示本机 AppKit 选择层，让未选区域暗下去并保持所选区域原始亮度。自定义区域开始录制后，Pulse 会继续显示一个不接收鼠标事件的本机区域指示层，让未录制区域保持暗化、所选区域保留蓝色边界；Pulse 会从最终录制输出中排除该指示层窗口。随后 Pulse 使用 macOS 原生 ScreenCaptureKit 录制选中的显示器、窗口或区域，并用 AVFoundation 写入 `.mov` 文件。“隐藏 pulse 界面”开启时，Pulse 会在目标选择前短暂隐藏；录制开始后 Pulse island 会恢复显示计时和停止按钮，但 ScreenCaptureKit 会从最终录制输出中排除 Pulse 自身，因此保存后的视频不会显示 Pulse island。录制期间 macOS 显示的系统共享指示由系统控制，Pulse 不会把它写入最终视频。录屏默认会隐藏鼠标指针；用户也可关闭“隐藏鼠标”，让指针出现在录制内容中。录屏会先保存到系统临时目录，停止后进入 island 视频预览；用户可以先用本机播放器预览，再选择保存、分享、关闭或丢弃。只有用户从保存面板选择位置后，Pulse 才会把临时 `.mov` 移动到该位置；关闭、丢弃、替换预览、退出或保存失败时，Pulse 会删除仍未保存的临时录屏文件。Pulse 不会把截图或录屏上传到网络。若权限尚未授予或已被拒绝，Pulse 会请求权限或打开系统隐私设置。

## Pulse 不会收集的数据

Pulse 不会收集、存储或传输：

- 个人文件
- 通讯录、照片、日历或定位数据
- 钥匙串内容；如果用户把密码、验证码、API key 或其他凭据复制到系统剪贴板，剪贴板模块会按历史记录本机保存，并在列表中默认遮罩
- 应用内行为分析数据或功能使用遥测
- 蓝牙地址、设备序列号或持久追踪标识的持久记录或上传
- 备忘内容、待办内容或完成状态的网络上传

## 网络使用

Pulse 会通过 HTTPS 请求 TimeLikeSilver 官网的 appcast 文件，用于检查是否存在新版本；当用户点击更新时，Pulse 会下载对应发布包并由 Sparkle 验证签名后安装。

和普通 HTTPS 请求一样，appcast 请求可能会由服务器处理标准请求元数据，例如 IP 地址、User-Agent、请求时间和请求路径。TimeLikeSilver 可能会将这些请求用于安全、运维和汇总更新检查次数，以了解 Pulse 的运行趋势。更新检查不会附加 CPU、内存、设备型号、系统启动时间、进程列表、应用程序列表、蓝牙设备列表、蓝牙电量、蓝牙充电状态、剪贴板内容、App bundle 路径、文件内容或 Sparkle 系统画像。

## 本地偏好设置

Pulse 可能会使用 macOS 本地存储保存语言、外观、开机启动、应用程序列表显示方式、常用应用 App bundle 路径、剪贴板 OCR 开关、剪贴板保留数量、剪贴板保留天数、截屏和录屏时是否隐藏 pulse 界面、录屏时是否隐藏鼠标，以及唤醒剪贴板、唤醒应用程序、三种截图模式和三种录屏模式的快捷键等偏好设置。备忘内容保存在本机 Application Support 目录，不写入用户偏好设置。蓝牙设备列表、连接状态、电量和充电状态不会写入本地偏好设置。
